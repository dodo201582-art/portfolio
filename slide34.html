아키텍처 설계부터 구축 · 이관까지 전 과정 수행

NHN Cloud 기반으로 KAIST LMS 학사 서비스 인프라를 아키텍처 설계부터 구축까지 진행했습니다. 기존에 타 환경에서 운영 중이던 KAIST 홈페이지를 NHN Cloud로 이관하여, LMS와 홈페이지 두 서비스를 단일 클라우드 환경에서 통합 운영하는 구조로 설계했습니다.

LMS 접속 시 홈페이지 리다이렉트 이슈 해결

구축 완료 후 첫 번째 이슈는 LMS 서비스 접속 시 홈페이지로 리다이렉트되는 문제였습니다. 원인을 추적한 결과 웹 서버 설정의 ServerAlias가 정확히 지정되지 않아 발생한 것으로 확인했고, 설정 수정으로 즉시 해결했습니다.

Redis 세션 공유 구간 패킷 이상 · WEB 서버 장애 분석

운영 중 학사 시스템에서 간헐적 부하와 함께 WEB 서버 전체가 정상 작동하지 않는 장애가 발생했습니다. WEB 서버에서는 원인을 찾지 못해 전체 구성을 재검토한 결과, Redis 서버에서 세션 공유 중 패킷이 정상 처리되지 않고 ESTABLISHED 상태가 대량으로 누적되고 있음을 확인했습니다.

개발팀 협업을 통한 애플리케이션단 근본 원인 해소

근본 원인은 개발 소스에서 데이터를 정상 처리하지 못하는 애플리케이션 문제였습니다. 개발팀과 협업하여 기능 개선 후 장애를 완전히 해소했습니다. 증상만 보면 인프라 문제로 보이지만 실제 원인은 애플리케이션단에 있었던 사례로, 인프라와 애플리케이션을 동시에 확인하여 장애를 해결한 경험이었습니다.
